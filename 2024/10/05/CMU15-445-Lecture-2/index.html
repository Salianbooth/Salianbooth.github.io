SQL History

  • SQL的前身是1971年IBM创建的SQUARE，随后演变为1972年的SEQUEL，最终发展为SQL。
  • 当前SQL标准是_SQL:2023_，它支持属性图查询和多维数组。之前的版本如SQL:2016引入了对JSON的支持，SQL:2011支持时间数据库等【8†source】。

SQL 分类

SQL可以基本分成以下的4类：

  1. Data Manipulation Language (DML)
  2. Data Definition Language (DDL)
  3. Data Query Language（DQL）
  4. Data Control Language (DCL)

    接下来我们分别介绍基本操作。

  5. SELECT：选取字段。
  6. ORDER BY：排序操作。
  7. LIMIT：分页查询。

多表查询

在你提供的PDF中，多表查询是通过使用多张表进行数据检索的操作。多表查询主要包括内连接、外连接（左连接、右连接）、交叉连接、子查询等。

1. 内连接（INNER JOIN）

  • 描述：内连接返回两张表中满足连接条件的记录。如果没有匹配的记录，结果集中不会包含该记录。
  • 语法：
  • 示例：查询所有订单及其对应的用户信息

2. 左连接（LEFT JOIN）

  • 描述：左连接会返回左表中的所有记录，即使右表中没有匹配的记录。对于没有匹配的记录，右表的字段值将显示为NULL。
  • 语法：
  • 示例：查询所有用户及其对应的订单信息（即使某些用户没有下订单）

3. 右连接（RIGHT JOIN）

  • 描述：右连接与左连接相反，返回右表中的所有记录，即使左表中没有匹配的记录。对于没有匹配的记录，左表的字段值将显示为NULL。
  • 语法：
  • 示例：查询所有订单及其对应的用户信息（即使某些订单没有对应的用户）

4. 交叉连接（CROSS JOIN）

  • 描述：交叉连接返回表1和表2的笛卡尔积，即每一行表1都会与每一行表2组合。交叉连接通常不使用ON条件。
  • 语法：
  • 示例：查询所有用户与所有订单的组合

5. 子查询（Subquery）

  • 描述：子查询是在查询中嵌套另一条查询语句。可以用于筛选数据、判断条件等。
  • 语法：
  • 示例：查询下了订单的用户信息

6. 联合查询（UNION）

  • 描述：UNION用于将两个或多个查询的结果合并为一个结果集，要求各查询返回的列数及数据类型相同。
  • 语法：
  • 示例：查询用户和管理员的姓名

总结

  • INNER JOIN 返回两张表中满足条件的记录。
  • LEFT JOIN 返回左表的所有记录，如果右表中没有匹配则显示NULL。
  • RIGHT JOIN 返回右表的所有记录，如果左表中没有匹配则显示NULL。
  • CROSS JOIN 返回表1和表2的笛卡尔积。
  • 子查询 允许查询嵌套其他查询。
  • UNION 合并多个查询结果。
